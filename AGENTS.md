# Project Instructions for Codex

This is a Google Apps Script project managed with clasp.

## Safety rules
- Do not run `clasp push` unless explicitly instructed.
- Do not run `git commit` unless explicitly instructed.
- Do not change `.clasp.json` unless explicitly instructed.
- Do not change `appsscript.json` unless necessary and explain why first.
- Before editing, read all relevant files and summarize the current flow.
- After editing, summarize changed files, changed functions, and possible risks.

## File roles
- `Code.js`: server-side Google Apps Script logic.
- `login.html`: login page UI.
- `index.html`: possible main dashboard UI.
- `index3.html`: possible alternate dashboard UI.
- `taskHistory.html`: task/history UI.
- `userPopup.html`: user popup UI.
- `appsscript.json`: Apps Script manifest.

## Required workflow
1. Analyze first.
2. Propose a plan.
3. Wait for approval.
4. Edit only approved files.
5. Do not push to Apps Script automatically.
